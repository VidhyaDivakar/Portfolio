## What is an API?

An API (Application Programming Interface) is a set of protocols that enables communication between software applications. Developed by software engineers, APIs allow systems built in different programming languages — C++, Java, Python, PHP, JavaScript — to exchange information securely and reliably.

Think of an API as a waiter in a restaurant. You (the client) tell the waiter (the API) what you want. The waiter takes your request to the kitchen (the server), and brings back exactly what was ordered. You never need to go into the kitchen yourself.

## Real-World Examples

**Online Payments**
When you click "Pay Now" on an e-commerce site, an API securely routes your payment information to a payment gateway, processes it, and returns a success or failure response — all within seconds.

**Weather Information**
Weather apps use third-party weather APIs to pull real-time data. The API retrieves current conditions, forecasts, and alerts from remote servers and delivers them to the app you're using.

**Location Services**
When you search for a business on Google Maps, an API fetches location data, business details, and routing information from backend servers and presents it in the interface.

## How APIs Work

APIs operate as intermediaries between clients and servers:

1. A user submits a request (clicks a button, enters a search term)
2. The client application sends that request to the API endpoint
3. The API transmits the request to the web server
4. The server processes the request and returns data
5. The API delivers that response back to the client

This request-response cycle is the foundation of how modern web applications communicate.

## What is API Documentation?

API documentation is a technical reference that explains how to use an API effectively. Like a user manual for software, it describes each endpoint, the data it accepts, the responses it returns, authentication methods, error codes, and usage examples.

Without good documentation, even a well-built API becomes difficult to use. Developers need to understand:
- What endpoints are available
- What parameters to pass
- What the response structure looks like
- How to handle errors

## The Technical Writer's Role

Technical writers collaborate with developers to understand API functionality, then translate that understanding into documentation that serves developers who will consume the API.

The documentation workflow typically follows this sequence:
1. New API requirements are defined
2. Developers build the API
3. Writers and developers collaborate to review behavior and edge cases
4. Writers produce the documentation
5. Documentation is reviewed, published, and maintained alongside API updates

## REST APIs

REST (Representational State Transfer) is the most widely used API architecture today. REST APIs are:
- **Language-agnostic** — usable from any programming language
- **Stateless** — each request contains all the information needed to process it
- **Standardised** — use HTTP methods (GET, POST, PUT, DELETE) that developers already understand

Most public APIs — from Twitter to GitHub to Stripe — are REST APIs, which makes REST API documentation one of the most in-demand technical writing skills in the industry.
